# PuzzleMind 🧠

**PuzzleMind** — это современное интерактивное веб-приложение на базе **.NET 10 Blazor Server**, предназначенное для автоматического поиска решений и визуализации популярных логических головоломок: **Water Sort** (сортировка жидкостей в пробирках) и **Mahjong / Triple Tile** (китайский пасьянс-маджонг).

Интерфейс приложения полностью построен на дизайн-системе **Radzen.Blazor** с использованием темы **Material**, что обеспечивает высокую скорость работы, полную адаптивность и отзывчивый UI.

---

## 🚀 Основные возможности

### 🧪 Water Sort Solver (Сортировка воды)
Интеллектуальный помощник для игры в сортировку цветной жидкости.
* **Гибкий конструктор доски:** Настройка любого количества пробирок (от 2 до 14) с сохранением текущих цветов при изменении структуры.
* **Интерактивный выбор:** Настройка цветов каждого слоя пробирки кликом мыши в режиме реального времени.
* **Плеер анимации решения:** Полноценный медиаплеер (Старт ▷, Пауза ⏸, Стоп ⏹) для пошагового просмотра алгоритма переливания.
* **Таймлайн ходов:** Интерактивная боковая панель со списком всех шагов на базе `RadzenTimeline`. Клик по шагу мгновенно перематывает доску на выбранное состояние.

### 🀄 Mahjong Solver (Редактор Triple Tile)
Инструмент для разбора сложных многослойных 3D-раскладок плиток Маджонга.
* **Послойный редактор:** Возможность добавлять плитки, настраивать их тип (ID картинки), слой (`Z-index`) и точные координаты на игровой доске.
* **Умный алгоритм разбора:** Расчет оптимальных комбинаций для полного очищения поля без захода в тупиковые ситуации.

## 🛠 Стек технологий

* **Платформа:** .NET 10.0 (ASP.NET Core)
* **Технология UI:** Blazor Server (Компонентный подход)
* **Режим рендеринга:** `InteractiveServer` (Высокая интерактивность на SignalR)
* **Библиотека компонентов:** Radzen.Blazor (Тема: *Material*)
* **Архитектурный паттерн:** Feature-Based Architecture (Изолированные модули в папке `Features/`)

## 📂 Структура проекта (Основные компоненты)

```text
├── Components/
│   ├── Layout/
│   │   ├── MainLayout.razor       # Главный каркас сайта (Сетка, Шапка, Боковое меню)
│   │   ├── NavMenu.razor          # Панель навигации на базе RadzenPanelMenu

│   │   └── ReconnectModal.razor.js# JS-логика управления состояниями сессии
│   ├── Pages/
│   │   ├── Home.razor             # Главная страница (Dashboard быстрого доступа)
│   │   ├── About.razor            # Страница описания возможностей приложения
│   │   ├── Error.razor            # Страница критических ошибок (500)
│   │   ├── NotFound.razor         # Страница ошибки 404
│   │   └── WaterSortSolverPage.razor # Локальный роут для запуска фичи сортировки
├── Features/
│   ├── WaterSortFeature.razor     # Изолированный интерактивный модуль сортировки воды
│   └── WaterSortFeature.razor.cs  # Code-Behind файл (бизнес-логика и плеер анимации)
└── Services/
    ├── WaterSortState.cs          # Модель состояния пробирок и хэширование графа
    └── WaterSolver.cs             # Алгоритм обхода в глубину (DFS) для поиска решения
```

## ⚙️ Настройка и запуск

### Требования
* **.NET 10.0 SDK** или выше.
* Среда разработки **Visual Studio 2025** / **JetBrains Rider** / **VS Code**.

### Установка и запуск локально
1. Клонируйте репозиторий с проектом:
   ```bash
   git clone https://github.com
   cd PuzzleMind
   ```
2. Убедитесь, что зависимости Radzen успешно восстановлены, и запустите проект:
   ```bash
   dotnet restore
   dotnet run --project WaterSortSolver
   ```
3. Откройте браузер и перейдите по адресу, указанному в консоли (обычно `http://localhost:5023`).

## 💡 Важные архитектурные решения

1. **Разделение разметки и кода (Code-Behind):** Крупные интерактивные блоки (например, `WaterSortFeature`) разделены на декларативную разметку `.razor` и C#-класс `.razor.cs`. Это упрощает чтение кода и ускоряет компиляцию.
2. **Feature-Driven подход:** Логика приложений не размазана по проекту, а инкапсулирована внутри папки `Features`. Страницы (Pages) выступают лишь точками входа (роутами), вызывая готовые модули.
3. **Отказоустойчивый ReconnectModal:** Окно потери связи с сервером использует нативный HTML5-тег `<dialog>`, что позволяет ему корректно отображаться и реагировать на действия пользователя даже в моменты полного падения бэкенда и SignalR-сессии.
